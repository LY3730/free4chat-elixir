# Free4Chat Docker 构建与部署 — 完整对话记录

## 项目概述

Free4Chat 是一个基于 WebRTC 的视频聊天应用：
- **后端**：Elixir 1.14 / Phoenix 1.6 / Membrane RTC Engine（端口 4000 TCP + TURN 50000-50050 UDP）
- **前端**：Next.js 12 / React 18 / TypeScript（端口 3000）
- **Docker Hub 镜像**：`hdknyg/free4chat`

---

## 问题一：Docker 无法拉取基础镜像

### 错误信息

```bash
docker build --build-arg DASHBOARD_AUTH_USERNAME=admin --build-arg DASHBOARD_AUTH_PASSWORD=password -t free4chat .
```

```
ERROR: failed to solve: alpine:3.16: failed to resolve source metadata for docker.io/library/alpine:3.16:
unable to fetch descriptor ... which reports content size of zero: invalid argument
```

### 原因

国内网络无法访问 Docker Hub (`docker.io`)，导致 `elixir:1.14.1-alpine` 和 `alpine:3.16` 两个基础镜像都无法拉取。

### 解决方案

在 Docker Desktop → Settings → Docker Engine 中配置镜像加速器：

```json
{
  "registry-mirrors": [
    "https://docker.1ms.run",
    "https://docker.xuanyuan.me"
  ]
}
```

点击 Apply & Restart 后重新构建，镜像拉取成功。

## 问题二：容器内无法访问 GitHub（Git 依赖拉取失败）

### 错误信息

```
ERROR [build 9/11] RUN mix setup
* Getting membrane_rtc_engine (https://github.com/membraneframework/membrane_rtc_engine.git)
fatal: unable to access 'https://github.com/membraneframework/membrane_rtc_engine.git/':
Failed to connect to github.com port 443 after 31 ms: Couldn't connect to server
```

### 原因

`mix.lock` 中 `membrane_rtc_engine` 使用的是 Git 仓库源而非 Hex 包：

```elixir
"membrane_rtc_engine": {:git, "https://github.com/membraneframework/membrane_rtc_engine.git", ...}
```

`mix setup` 执行时会调用 `git clone`，但容器内无法访问 GitHub。

### 尝试的方案

#### 方案 A：GitHub 代理（不稳定）

1. **ghproxy.net** — 报 `PROTOCOL_ERROR`
2. **mirror.ghproxy.com** — 同样不稳定

均失败。

#### 方案 B：本地克隆 + Path 依赖（最终成功）

**核心思路**：提前在本地把 GitHub 仓库下载好，Docker 构建时直接从本地读取。

##### 步骤 1：本地克隆

```bash
cd backend
git clone https://github.com/membraneframework/membrane_rtc_engine.git deps/membrane_rtc_engine
```

##### 步骤 2：修改 `backend/mix.exs`（第 43 行）

```elixir
# 修改前
{:membrane_rtc_engine, github: "membraneframework/membrane_rtc_engine", override: true},

# 修改后
{:membrane_rtc_engine, path: "deps/membrane_rtc_engine", override: true},
```

将 Git 远程依赖改为本地路径依赖。

##### 步骤 3：修改 `backend/Dockerfile`

移除 git 代理配置行，添加 COPY 指令：

```dockerfile
COPY lib lib
# copy local membrane_rtc_engine (pre-cloned from GitHub, no network needed)
COPY deps/membrane_rtc_engine deps/membrane_rtc_engine

RUN mix setup
RUN mix deps.compile
```

##### 步骤 4：修改 `backend/.dockerignore`

默认排除了 `/deps/`，需要放行：

```
/deps/
!/deps/membrane_rtc_engine
```

##### 结果

```
[+] Building 95.9s (23/23) FINISHED
 => naming to docker.io/library/free4chat:latest
```

## 问题三：运行容器缺少必要环境变量

### 错误信息

```bash
docker run -p 50000-50050:50000-50050/udp -p 4000:4000/tcp \
  -e INTEGRATED_TURN_PORT_RANGE=50000-50050 \
  -e INTEGRATED_TURN_IP=127.0.0.1 \
  free4chat
```

```
(System.EnvError) could not fetch environment variable "VIRTUAL_HOST" because it is not set
```

### 原因

Phoenix 的 `runtime.exs` 通过 `System.fetch_env!/1` 强制要求多个环境变量。

### 解决方案

运行时传入所有必需变量：

```bash
docker run -d --name free4chat --hostname free4chat --restart unless-stopped \
  -p 4000:4000/tcp \
  -p 50000-50050:50000-50050/udp \
  -e DASHBOARD_AUTH_USERNAME=admin \
  -e DASHBOARD_AUTH_PASSWORD=password \
  -e VIRTUAL_HOST=localhost \
  -e USE_TLS=false \
  -e SECRET_KEY_BASE=free4chat_secret_key_base_2024 \
  -e INTEGRATED_TURN_IP=127.0.0.1 \
  -e INTEGRATED_TURN_PORT_RANGE=50000-50050 \
  free4chat
```

## 问题四：Erlang 节点名非法导致 Phoenix 未启动

### 错误信息

```
11:01:44.021 [warning] Integrated TURN certificate or private key path not specified.

11:01:44.711 [error] ** System NOT running to use fully qualified hostnames **
** Hostname 172.26.8.68 is illegal **

11:01:44.711 [warning] [libcluster:free4chat] unable to connect to :"rtc1@172.26.8.68"
```

### 原因

`runtime.exs` 中 libcluster 配置硬编码了生产服务器 IP：

```elixir
config :libcluster,
  topologies: [
    free4chat: [
      strategy: Cluster.Strategy.Epmd,
      config: [hosts: [:"rtc1@172.26.8.68"]]   # 硬编码的生产 IP
    ]
  ]
```

Docker 容器内网 IP 不匹配，Erlang 分布式系统启动异常，导致 Phoenix Endpoint 没有监听 4000 端口。

### 解决方案

修改 `backend/config/runtime.exs`，本地开发时跳过集群配置：

```elixir
# Only configure clustering for non-localhost environments
if host != "localhost" do
  config :libcluster,
    debug: true,
    topologies: [
      free4chat: [
        strategy: Cluster.Strategy.Epmd,
        config: [hosts: [:"rtc1@#{host}"]]
      ]
    ]
end
```

同时添加 `--hostname free4chat` 参数确保节点名合法。

## 问题五：PHX_SERVER 环境变量丢失

### 现象

容器启动后 Phoenix 没有监听 4000 端口。检查发现：

```
PHX_SERVER=          # 空！
MIX_ENV=            # 空！
```

### 原因

Dockerfile 使用多阶段构建：

- **构建阶段** (`FROM elixir:1.14.1-alpine AS build`)：第 23 行设置了 `ENV PHX_SERVER=TRUE`
- **运行阶段** (`FROM alpine:3.16 AS app`)：**没有继承**该环境变量

`PHX_SERVER=TRUE` 是 Phoenix 启动 HTTP 服务器的关键标志，缺失导致只启动了 Erlang VM 而没有启动 Web 服务器。

### 解决方案

在 `Dockerfile` 的 app 阶段补充环境变量：

```dockerfile
# prepare release image
FROM alpine:3.16 AS app

# install runtime dependencies
RUN apk add --no-cache openssl ncurses-libs libsrtp ffmpeg clang curl

WORKDIR /app

ENV PHX_SERVER=TRUE        # ← 新增这行
```

## 问题六：WebSocket 跨域 Origin 检查失败

### 错误信息（前端浏览器）

```
Unhandled Runtime Error
Error: tried to push 'mediaEvent' to 'room:xxx' before joining.
Use channel.join() before pushing events
```

### 后端日志

```
Origin of the request: http://localhost:3000

This happens when you are attempting a socket connection to
a different host than the one configured in your config/files...
```

### 原因

前端在 `localhost:3000`，后端在 `localhost:4000`。`prod.exs` 只允许 `*.free4.chat` 来源：

```elixir
config :free4chat, Free4chatWeb.Endpoint,
  check_origin: ["//*.free4.chat"]
```

### 解决方案

在 `runtime.exs` 中覆盖 check_origin 配置：

```elixir
config :free4chat, Free4chatWeb.Endpoint, [
  {:url, [host: host]},
  {protocol, args},
  check_origin: (host == "localhost" && false) || ["//#{host}"]
]
```

本地开发时允许所有来源，生产环境只允许配置域名。

同时修改前端 `frontend/src/common/consts.tsx` 连接地址：

```typescript
const serverAddrs = [
  "ws://localhost:4000/socket",   // 本地用 ws://
]
```

## 问题七：Hex 包仓库下载超时

### 错误信息

```
* Getting phoenix_live_dashboard (Hex package)
Request failed (:timeout)
If this happens consistently, adjust your concurrency and timeout settings:
    HEX_HTTP_CONCURRENCY=1 HEX_HTTP_TIMEOUT=120 mix deps.get
(Mix) Package fetch failed and no cached copy available (https://repo.hex.pm/tarballs/...)
```

### 原因

Docker 容器内访问官方 Hex 包仓库 `repo.hex.pm` 超时（国内网络问题）。

### 尝试的方案

#### 方案 A：hex.config 文件（语法错误 x2）

1. 第一次写法：`mirror: https://hexpm.upyun.com` → URL 缺引号，SyntaxError
2. 第二次写法：`mirror: "https://hexpm.upyun.com"` → 键名错误，syntax error before: mirror
3. 第三次写法：`%{hex_mirror: "https://hexpm.upyun.com"}` → 文件格式不被识别

#### 方案 B：环境变量（成功）

在 `Dockerfile` 中使用环境变量方式：

```dockerfile
# configure Hex mirror + timeout (for China network)
ENV HEX_MIRROR=https://hexpm.upyun.com
ENV HEX_HTTP_CONCURRENCY=1 HEX_HTTP_TIMEOUT=120
```

重新构建后成功通过又云镜像下载所有包。

## 最终可用的 Dockerfile

```dockerfile
FROM elixir:1.14.1-alpine AS build

RUN apk add --no-cache \
  build-base npm git python3 make cmake openssl-dev libsrtp-dev ffmpeg-dev clang-dev

ARG DASHBOARD_AUTH_USERNAME
ENV DASHBOARD_AUTH_USERNAME=${DASHBOARD_AUTH_USERNAME}

ARG DASHBOARD_AUTH_PASSWORD
ENV DASHBOARD_AUTH_PASSWORD=${DASHBOARD_AUTH_PASSWORD}

ENV PHX_SERVER=TRUE

WORKDIR /app

# install hex + rebar + Hex mirror config
RUN mix local.hex --force && \
  mix local.rebar --force

# configure Hex mirror + timeout (for China network)
ENV HEX_MIRROR=https://hexpm.upyun.com
ENV HEX_HTTP_CONCURRENCY=1 HEX_HTTP_TIMEOUT=120

ENV MIX_ENV=prod

COPY mix.exs mix.lock ./
COPY config config
COPY priv priv
COPY lib lib
COPY deps/membrane_rtc_engine deps/membrane_rtc_engine

RUN mix setup
RUN mix deps.compile

RUN mix do compile, release

FROM alpine:3.16 AS app

RUN apk add --no-cache openssl ncurses-libs libsrtp ffmpeg clang curl

WORKDIR /app

ENV PHX_SERVER=TRUE

RUN chown nobody:nobody /app

USER nobody:nobody

COPY --from=build --chown=nobody:nobody /app/_build/prod/rel/free4chat ./

ENV HOME=/app

EXPOSE 4000

HEALTHCHECK CMD curl --fail http://localhost:4000/api/healthcheck || exit 1

CMD ["bin/free4chat", "start"]
```

## 完整操作流程总结

### 本地构建

```bash
cd backend

# 1. 克隆 membrane_rtc_engine（仅需一次）
git clone https://github.com/membraneframework/membrane_rtc_engine.git deps/membrane_rtc_engine

# 2. 构建
docker build --build-arg DASHBOARD_AUTH_USERNAME=admin --build-arg DASHBOARD_AUTH_PASSWORD=password -t free4chat .

# 3. 运行
docker run -d --name free4chat --hostname free4chat --restart unless-stopped \
  -p 4000:4000/tcp -p 50000-50050:50000-50050/udp \
  -e DASHBOARD_AUTH_USERNAME=admin -e DASHBOARD_AUTH_PASSWORD=password \
  -e VIRTUAL_HOST=localhost -e USE_TLS=false \
  -e SECRET_KEY_BASE=your_secret_key -e INTEGRATED_TURN_IP=127.0.0.1 \
  -e INTEGRATED_TURN_PORT_RANGE=50000-50050 \
  free4chat
```

### 前端启动

```bash
cd frontend && yarn install && yarn dev
# 访问 http://localhost:3000
```

### 推送到 Docker Hub

```bash
docker tag free4chat:latest hdknyg/free4chat:latest
docker push hdknyg/free4chat:latest
```

### 服务器部署

```bash
docker pull hdknyg/free4chat:latest
docker run -d --name free4chat --restart unless-stopped \
  -p 4000:4000/tcp -p 50000-50050:50000-50050/udp \
  -e DASHBOARD_AUTH_USERNAME=admin -e DASHBOARD_AUTH_PASSWORD=你的密码 \
  -e VIRTUAL_HOST=你的域名或IP -e USE_TLS=false \
  -e SECRET_KEY_BASE=$(openssl rand -base64 64) \
  -e INTEGRATED_TURN_IP=服务器公网IP \
  -e INTEGRATED_TURN_PORT_RANGE=50000-50050 \
  hdknyg/free4chat
```

## 经验教训

1. **Elixir Mix 项目的 Git 依赖** 在 Docker 构建中是常见痛点，`path` 依赖是最可靠的绕过方案。
2. **`.dockerignore`** 的 `/deps/` 排除规则会影响手动 COPY 的依赖，需注意放行。
3. **Docker 多阶段构建**中，`ENV` 变量不会跨阶段继承，每个 FROM 都需要重新设置关键变量（如 `PHX_SERVER`）。
4. **Hex 镜像配置** 用环境变量（`HEX_MIRROR`）比 `.hex/hex.config` 文件更可靠。
5. **Phoenix 生产环境** 通过 `runtime.exs` 严格管理配置，本地开发和生产环境的差异需要条件判断处理。
6. **Erlang 分布式系统** 对主机名敏感，Docker 容器必须设置合理的 `--hostname`。
7. **国内网络环境** 下，Docker Hub、GitHub、Hex 三层都需要分别解决访问问题。
