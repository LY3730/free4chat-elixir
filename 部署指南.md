# Free4Chat 部署指南

## 项目架构

```
free4chat-elixir/
├── backend/          # Elixir/Phoenix 后端（WebRTC 视频通话 + WebSocket）
├── frontend/         # Next.js 前端
└── infra/            # 部署脚本和基础设施配置
```

| 组件 | 技术栈 | 端口 |
|------|--------|------|
| 后端 | Elixir 1.14 / Phoenix 1.6 / Membrane RTC Engine | 4000 (TCP) |
| TURN 服务 | 内置于后端，用于 NAT 穿越 | 50000-50050 (UDP) |
| 前端 | Next.js 12 / React 18 / TypeScript | 3000 (开发) |

---

## 一、本地开发运行

### 1. 后端（需要 Docker）

```bash
cd backend

# 第 1 步：克隆 membrane_rtc_engine（国内 GitHub 访问受限，需提前下载）
git clone https://github.com/membraneframework/membrane_rtc_engine.git deps/membrane_rtc_engine

# 第 2 步：构建 Docker 镜像
docker build --build-arg DASHBOARD_AUTH_USERNAME=admin --build-arg DASHBOARD_AUTH_PASSWORD=password -t free4chat .

# 第 3 步：运行容器
docker run -d --name free4chat \
  -p 4000:4000/tcp \
  -p 50000-50050:50000-50050/udp \
  -e DASHBOARD_AUTH_USERNAME=admin \
  -e DASHBOARD_AUTH_PASSWORD=password \
  -e VIRTUAL_HOST=localhost \
  -e USE_TLS=false \
  -e SECRET_KEY_BASE=your_secret_key_here \
  -e INTEGRATED_TURN_IP=127.0.0.1 \
  -e INTEGRATED_TURN_PORT_RANGE=50000-50050 \
  free4chat
```

访问：
- 应用地址：http://localhost:4000
- Dashboard：用户名 `admin`，密码 `password`

### 2. 前端

```bash
cd frontend

# 安装依赖
yarn install

# 开发模式启动（默认端口 3000）
yarn dev

# 或者先构建再以生产模式启动
yarn build
yarn start
```

访问：http://localhost:3000

## 二、服务器部署（Linux）

### 前提条件

- 服务器已安装 Docker

> **镜像已推送至 Docker Hub：`hdknyg/free4chat`**，服务器上可以直接拉取运行，无需从源码构建。

### 方式 A：直接拉取镜像运行（推荐）

```bash
# 拉取镜像
docker pull hdknyg/free4chat

# 运行容器
docker run -d --name free4chat --restart unless-stopped \
  -p 4000:4000/tcp \
  -p 50000-50050:50000-50050/udp \
  -e DASHBOARD_AUTH_USERNAME=admin \
  -e DASHBOARD_AUTH_PASSWORD=你的密码 \
  -e VIRTUAL_HOST=你的域名或IP \
  -e USE_TLS=false \
  -e SECRET_KEY_BASE=$(openssl rand -base64 64) \
  -e INTEGRATED_TURN_IP=服务器公网IP \
  -e INTEGRATED_TURN_PORT_RANGE=50000-50050 \
  hdknyg/free4chat
```

### 方式 B：从源码构建

如果需要自定义修改后重新构建：

#### 1. 初始化服务器（可选）

```bash
# 使用项目自带脚本初始化服务器环境
./infra/scripts/init-server.sh
```

#### 2. 上传代码到服务器

```bash
# 在本地打包（排除 node_modules、_build 等大目录）
tar czf free4chat.tar.gz --exclude='frontend/node_modules' --exclude='backend/deps' --exclude='backend/_build' --exclude='.git' free4chat-elixir/

# 上传到服务器
scp free4chat.tar.gz user@your-server:/home/user/
```

在服务器上：

```bash
cd /home/user
tar xzf free4chat.tar.gz
cd free4chat-elixir/backend
```

#### 3. 克隆依赖并构建镜像

```bash
# 克隆 membrane_rtc_engine（如果服务器也无法访问 GitHub，需从本地上传）
git clone https://github.com/membraneframework/membrane_rtc_engine.git deps/membrane_rtc_engine

# 构建镜像
docker build --build-arg DASHBOARD_AUTH_USERNAME=admin --build-arg DASHBOARD_AUTH_PASSWORD=your_secure_password -t free4chat .
```

#### 4. 运行容器

```bash
docker run -d --name free4chat --restart unless-stopped \
  -p 4000:4000/tcp \
  -p 50000-50050:50000-50050/udp \
  -e DASHBOARD_AUTH_USERNAME=admin \
  -e DASHBOARD_AUTH_PASSWORD=your_secure_password \
  -e VIRTUAL_HOST=你的域名或IP \
  -e USE_TLS=false \
  -e SECRET_KEY_BASE=$(openssl rand -base64 64) \
  -e INTEGRATED_TURN_IP=服务器公网IP \
  -e INTEGRATED_TURN_PORT_RANGE=50000-50050 \
  free4chat
```

> **重要**: `INTEGRATED_TURN_IP` 必须设置为服务器的**公网 IP**，否则 WebRTC 的 TURN 中继无法正常工作。

### 5. 配置防火墙

需要在云服务器的安全组/防火墙中开放以下端口：

| 协议 | 端口 | 用途 |
|------|------|------|
| TCP | 4000 | Web 应用 |
| TCP | 80 | HTTP（如需重定向） |
| TCP | 443 | HTTPS（如需 TLS） |
| UDP | 50000-50050 | TURN 中继 |

### 6. 端口转发（可选，将 80/443 转发到 4000）

```bash
sudo iptables -t nat -A PREROUTING -p tcp --dport 80 -j REDIRECT --to-port 4000
sudo iptables -t nat -A PREROUTING -p tcp --dport 443 -j REDIRECT --to-port 4000
```

> 注意：iptables 规则重启后会丢失，需要重新执行或配置持久化。

### 7. 启用 TLS/HTTPS（可选）

如果需要 HTTPS，修改运行参数：

```bash
-e USE_TLS=true \
-e KEY_FILE_PATH=/path/to/key.pem \
-e CERT_FILE_PATH=/path/to/cert.pem \
```

并将证书文件通过 `-v` 卷挂载进容器。

## 四、前端部署（Vercel 推荐）

前端推荐使用 [Vercel](https://vercel.com) 一键部署：

1. 将 `frontend` 目录连接到 Vercel 项目
2. Vercel 会自动检测 Next.js 并构建部署
3. 在 Vercel 设置中配置后端 API 地址的环境变量

也可以使用其他平台（Netlify、自建 Nginx 等），只需 `yarn build` 后托管 `.next` 目录或使用 `yarn start`。

## 六、环境变量说明

| 变量名 | 必须 | 说明 |
|--------|------|------|
| `DASHBOARD_AUTH_USERNAME` | 是 | 管理 Dashboard 用户名 |
| `DASHBOARD_AUTH_PASSWORD` | 是 | 管理 Dashboard 密码 |
| `VIRTUAL_HOST` | 是 | 服务器主机名（域名或 IP） |
| `USE_TLS` | 否 | 是否启用 TLS（默认 false） |
| `SECRET_KEY_BASE` | 是 | Phoenix 加密密钥（用 `mix phx.gen.secret` 生成） |
| `INTEGRATED_TURN_IP` | 是 | TURN 服务器公网 IP |
| `INTEGRATED_TURN_PORT_RANGE` | 是 | TURN UDP 端口范围 |
| `KEY_FILE_PATH` | 否 | TLS 私钥路径 |
| `CERT_FILE_PATH` | 否 | TLS 证书路径 |
