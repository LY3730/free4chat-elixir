## 1. 架构概览

```
用户浏览器
    │
    ▼ (HTTPS:443, 自签名证书，需手动接受)
┌──────────────────────────────┐
│         Nginx                │
│   ┌───────────┬──────────┐   │
│   │  /        │ /socket  │   │
│   │  → :3000  │ → :4000  │   │
│   └───────────┴──────────┘   │
└──────────────────────────────┘
         │              │
         ▼              ▼
  ┌──────────┐   ┌──────────────┐
  │ Next.js  │   │ Docker       │
  │ 前端页面  │   │ Phoenix 后端 │
  │ (:3000)  │   │ (:4000 HTTPS)│
  └──────────┘   └──────────────┘
                       │
                  TURN UDP
               (:50000-50050)
```

| 端口 | 服务 | 说明 |
|------|------|------|
| 443/TCP | Nginx | 对外唯一入口（HTTPS），代理前端+WebSocket |
| 3000/TCP | Next.js | 前端页面（仅内网访问） |
| 4000/TCP | Docker/Phoenix | 后端 API + WebSocket（仅内网访问） |
| 50000-50050/UDP | Docker/TURN | WebRTC NAT 穿透（需开放） |

## 2. 前置准备

### 2.1 安装基础工具

```bash
apt-get update && apt-get install -y \
  docker.io nginx curl openssl git wget unzip
```
###配置docker镜像
```bash
sudo mkdir -p /etc/docker
sudo tee /etc/docker/daemon.json <<-'EOF'
{
  "registry-mirrors": [
    "https://docker-0.unsee.tech",
    "https://docker-cf.registry.cyou",
    "https://docker.1panel.live"
  ]
}
EOF
```
### 2.2 启动 Docker

```bash
systemctl enable --now docker
```

### 2.3 拉取镜像

```bash
docker pull hdknyg/free4chat:latest
```

### 2.4 获取服务器 IP

```bash
curl -4 ifconfig.me
```
> 记下这个 IP，后续步骤中用 `<YOUR_SERVER_IP>` 替代。

## 3. 生成 SSL 证书

由于没有域名，使用自签名证书。浏览器会显示安全警告，需要手动点击"继续前往"。

```bash
mkdir -p ~/certs && cd ~/certs

openssl req -x509 -nodes -days 365 -newkey rsa:2048 \
  -keyout key.pem \
  -out certificate.pem \
  -subj "/CN=<YOUR_SERVER_IP>" \
  -addext "subjectAltName=IP:<YOUR_SERVER_IP>"
```

### 设置正确权限（重要！）

```bash
chmod 644 ~/certs/key.pem ~/certs/certificate.pem
```

> ⚠️ 如果权限不对，Docker 容器内的 `nobody` 用户读不了私钥文件，TLS 握手将失败。

## 4. 启动后端 (Docker)

### 4.1 运行容器

```bash
docker run -d --name free4chat --hostname free4chat --restart unless-stopped \
  -p 4000:4000/tcp \
  -p 50000-50050:50000-50050/udp \
  -e DASHBOARD_AUTH_USERNAME=admin \
  -e DASHBOARD_AUTH_PASSWORD='YourPasswordHere' \
  -e VIRTUAL_HOST=<YOUR_SERVER_IP> \
  -e USE_TLS=true \
  -v ~/certs:/certs:ro \
  -e KEY_FILE_PATH=/certs/key.pem \
  -e CERT_FILE_PATH=/certs/certificate.pem \
  -e SECRET_KEY_BASE="$(openssl rand -base64 64)" \
  -e INTEGRATED_TURN_IP=<YOUR_SERVER_IP> \
  -e INTEGRATED_TURN_PORT_RANGE=50000-50050 \
  hdknyg/free4chat:latest
```

### 4.2 验证后端运行状态

```bash
# 检查容器是否在运行
docker ps | grep free4chat

# 检查日志是否正常启动
docker logs --tail 10 free4chat

# 验证 HTTPS 是否工作（应返回 "ok"）
curl -sk --connect-timeout 5 https://127.0.0.1:4000/api/healthcheck
```

正常输出示例：
```
<your_ip>: Running Free4chatWeb.Endpoint with cowboy 2.9.0 at 0.0.0.0:4000 (https)
ok
```

## 6. 配置前端 WebSocket 地址

修改 `frontend/src/common/consts.tsx`，将 WebSocket 地址指向服务器：

```bash
sed -i 's|"ws://localhost:4000/socket"|"wss://<YOUR_SERVER_IP>/socket"|' /opt/free4chat-elixir/frontend/src/common/consts.tsx
```

验证修改：

```bash
grep -A2 "serverAddrs" /opt/free4chat-elixir/frontend/src/common/consts.tsx
```

应该输出：

```
const serverAddrs = [
    "wss://<YOUR_SERVER_IP>/socket",
```

> 注意：这里**没有端口号**！WebSocket 走 Nginx 的 443 端口，由 Nginx 转发到后端 4000。
>
> 为什么不走 `wss://IP:4000`？因为浏览器的安全策略——用户只接受过 443 端口的证书，4000 端口的自签名证书会被直接拒绝。

## 8. 配置 Nginx 反向代理

Nginx 是整个系统的入口，负责：
1. **HTTPS 终结**：处理 SSL 证书
2. **前端页面代理**：`/` → Next.js `:3000`
3. **WebSocket 代理**：`/socket` → Phoenix `:4000`

### 8.1 创建 Nginx 配置文件

```bash
cat > /etc/nginx/sites-available/free4chat << 'EOF'
server {
    listen 443 ssl;
    server_name <YOUR_SERVER_IP>;

    # SSL 证书路径
    ssl_certificate     /root/certs/certificate.pem;
    ssl_certificate_key /root/certs/key.pem;

    # 前端页面 -> Next.js :3000
    location / {
        proxy_pass http://127.0.0.1:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

    # WebSocket -> 后端 Docker :4000
    location /socket {
        proxy_pass https://127.0.0.1:4000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_ssl_verify off;  # 关键：不验证后端自签名证书
    }
}
EOF
```

### 8.2 激活配置

```bash
ln -sf /etc/nginx/sites-available/free4chat /etc/nginx/sites-enabled/
rm -f /etc/nginx/sites-enabled/default
nginx -t && systemctl reload nginx
```

### 8.3 验证 Nginx 工作正常

```bash
# 测试 HTTPS 访问（应返回前端页面）
curl -sk https://127.0.0.1/ | head -5

# 测试 WebSocket 代理（应返回 101 Switching Protocols）
curl -sk -i -N -H "Connection: Upgrade" -H "Upgrade: websocket" -H "Host: <YOUR_SERVER_IP>" https://127.0.0.1/socket/websocket 2>&1 | head -15
```

## 9. 验证部署

### 9.1 浏览器测试

1. 打开浏览器，访问 **`https://<YOUR_SERVER_IP>`**
2. 浏览器会弹出**安全警告**（因为自签名证书）
3. 点击 **「高级」→ 「继续前往 <YOUR_SERVER_IP>(不安全)」**
4. 进入首页后，输入：
   - **Room Name（房间名）**: 任意名称，如 `test`
   - **Nick Name（昵称）**: 你的名字
5. 点击进入房间
6. 浏览器会弹出**麦克风权限请求**，点击「允许」
7. 如果能看到你的用户卡片和麦克风按钮 → **部署成功！**

### 9.2 多人测试

用另一台设备/浏览器打开同一个 URL：
- 输入**相同**的 Room Name（房间名）
- 输入**不同**的 Nick Name（昵称）
- 两边应该都能看到对方的声音波形

## 10. 防火墙与安全

### 10.1 开放必要端口

```bash
ufw allow 443/tcp    # HTTPS/Nginx 入口
ufw allow 22/tcp     # SSH
ufw allow 50000:50050/udp  # WebRTC TURN
ufw enable
```

> 注意：**不需要**开放 3000 和 4000 端口给外网！它们只通过 Nginx 内部转发。

### 10.2 安全建议

| 建议 | 说明 |
|------|------|
| 更改 Dashboard 密码 | `-e DASHBOARD_AUTH_PASSWORD=强密码` |
| 使用域名 + Let's Encrypt | 免费正规证书，无需浏览器警告 |
| 使用 Cloudflare Tunnel | 不暴露真实 IP，免费 CDN 加速 |
| 限制 IP 访问 | 在防火墙层面限制来源 IP |

## 11. 常用运维命令

### 后端相关

```bash
# 查看后端日志
docker logs -f free4chat

# 重启后端
docker restart free4chat

# 停止/删除
docker stop free4chat && docker rm free4chat

# 更新到最新镜像
docker pull hdknyg/free4chat:latest
docker stop free4chat && docker rm free4chat
# 然后重新执行第 4 步的 docker run 命令
```

### 前端相关

```bash
# 查看前端进程
ps aux | grep "next start"

# 重启前端
pkill -f "next start"
cd /opt/free4chat-elixir/frontend && nohup npm run start -- -p 3000 > /tmp/frontend.log 2>&1 &

# 重新构建（修改代码后）
pkill -f "next start"
cd /opt/free4chat-elixir/frontend && npm run build && nohup npm run start -- -p 3000 > /tmp/frontend.log 2>&1 &
```

### Nginx 相关

```bash
# 重载配置（不中断连接）
nginx -t && systemctl reload nginx

# 查看 Nginx 日志
tail -f /var/log/nginx/error.log
```

### 全部重启顺序

```bash
# 1. 重启后端
docker restart free4chat

# 2. 重启前端
pkill -f "next start"
cd /opt/free4chat-elixir/frontend && nohup npm run start -- -p 3000 > /tmp/frontend.log 2>&1 &

# 3. 重载 Nginx（通常不需要）
nginx -t && systemctl reload nginx
```

## 12. 故障排查

### 问题 1：后端 TLS 握手失败 (`unexpected eof while reading`)

**症状**: `curl -sk https://127.0.0.1:4000/` 报 SSL 错误

| 可能原因 | 解决方法 |
|----------|----------|
| 证书目录不存在 | 确认 `-v` 参数挂载路径正确，`ls /certs/` |
| 私钥权限不够 | `chmod 644 key.pem certificate.pem` |
| 文件名错误 | 确认 `CERT_FILE_PATH` 和实际文件名一致 |

### 问题 2：前端连不上后端 WebSocket (`WebSocket connection failed`)

**症状**: 浏览器控制台报 `wss://... failed`

| 可能原因 | 解决方法 |
|----------|----------|
| 前端地址还是旧的 `ws://localhost:4000` | 重新 `sed` 修改 + `npm run build` |
| Nginx 没有 `/socket` 代理规则 | 更新 `/etc/nginx/sites-available/free4chat` 并 reload |
| 前端没重新构建 | `consts.tsx` 是编译时打包的，必须 rebuild |

### 问题 3：浏览器无法申请麦克风

**症状**: `getUserMedia` 报错或无弹窗

| 可能原因 | 解决方法 |
|----------|----------|
| 用 HTTP 而非 HTTPS 访问 | 必须用 `https://` 访问 |
| 没有接受自签名证书警告 | 点「高级」→「继续前往」 |
| 浏览器不支持 | Chrome/Firefox/Safari 均支持 |

### 问题 4：Erlang 主机名错误 (`Hostname ... is illegal`)

**症状**: Docker 容器无法启动

| 可能原因 | 解决方法 |
|----------|----------|
| `VIRTUAL_HOST` 是 IP 地址 | 已修复，runtime.exs 会自动跳过 IP 的集群配置 |
| `VIRTUAL_HOST` 是 localhost | 同上 |

### 问题 5：Phoenix 不监听端口

**症状**: 容器运行中但 `netstat` 无 4000 端口

| 可能原因 | 解决方法 |
|----------|----------|
| 缺少 `PHX_SERVER=TRUE` | 镜像已包含此 ENV，确保用的是新镜像 |

### 问题 6：Nginx 502 Bad Gateway

**症状**: 访问 `https://<IP>` 显示 502

| 可能原因 | 解决方法 |
|----------|----------|
| 前端未启动 | `cd frontend && npm run start -- -p 3000 &` |
| 前端还在构建中 | 等 `npm run build` 完成 |
| 端口不对 | 确认 Next.js 监听的是 3000 端口 |

## 附录 A：一键部署脚本（可选）

如果想在全新 Ubuntu 上快速部署，可以创建以下脚本：

```bash
#!/bin/bash
set -e

# ========== 配置区（修改这里的值）==========
SERVER_IP="<YOUR_SERVER_IP>"
DASH_PASSWORD="YourSecurePassword"
# =========================================

echo "[1/8] 安装基础工具..."
apt-get update && apt-get install -y docker.io nginx curl openssl nodejs npm ufw
systemctl enable --now docker

echo "[2/8] 生成 SSL 证书..."
mkdir -p ~/certs && cd ~/certs
openssl req -x509 -nodes -days 365 -newkey rsa:2048 \
  -keyout key.pem -out certificate.pem \
  -subj "/CN=${SERVER_IP}" \
  -addext "subjectAltName=IP:${SERVER_IP}"
chmod 644 key.pem certificate.pem

echo "[3/8] 启动后端 Docker..."
docker run -d --name free4chat --hostname free4chat --restart unless-stopped \
  -p 4000:4000/tcp -p 50000-50050:50000-50050/udp \
  -e DASHBOARD_AUTH_USERNAME=admin -e DASHBOARD_AUTH_PASSWORD="${DASH_PASSWORD}" \
  -e VIRTUAL_HOST="${SERVER_IP}" -e USE_TLS=true \
  -v ~/certs:/certs:ro \
  -e KEY_FILE_PATH=/certs/key.pem -e CERT_FILE_PATH=/certs/certificate.pem \
  -e SECRET_KEY_BASE="$(openssl rand -base64 64)" \
  -e INTEGRATED_TURN_IP="${SERVER_IP}" -e INTEGRATED_TURN_PORT_RANGE=50000-50050 \
  hdknyg/free4chat:latest

echo "[4/8] 等待后端启动..."
sleep 5
curl -sk https://127.0.0.1:4000/api/healthcheck || { echo "后端启动失败!"; exit 1; }

echo "[5/8] 安装 Node.js 并准备前端代码..."
# 请提前上传代码到 /opt/free4chat-elixir/
cd /opt/free4chat-elixir/frontend
sed -i "s|ws://localhost:4000/socket|wss://${SERVER_IP}/socket|" src/common/consts.tsx
npm install && npm run build

echo "[6/8] 启动前端..."
nohup npm run start -- -p 3000 > /tmp/frontend.log 2>&1 &
sleep 3

echo "[7/8] 配置 Nginx..."
cat > /etc/nginx/sites-available/free4chat << NGINX_EOF
server {
    listen 443 ssl; server_name ${SERVER_IP};
    ssl_certificate     /root/certs/certificate.pem;
    ssl_certificate_key /root/certs/key.pem;

    location / {
        proxy_pass http://127.0.0.1:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade \$http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_set_header Host \$host;
        proxy_set_header X-Real-IP \$remote_addr;
    }

    location /socket {
        proxy_pass https://127.0.0.1:4000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade \$http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_set_header Host \$host;
        proxy_set_header X-Real-IP \$remote_addr;
        proxy_ssl_verify off;
    }
}
NGINX_EOF

ln -sf /etc/nginx/sites-available/free4chat /etc/nginx/sites-enabled/
rm -f /etc/nginx/sites-enabled/default
nginx -t && systemctl reload nginx

echo "[8/8] 配置防火墙..."
ufw allow 443/tcp && ufw allow 22/tcp && ufw allow 50000:50050/udp
# ufw enable  # 取消注释以启用防火墙

echo ""
echo "=========================================="
echo "  部署完成！"
echo "  访问地址: https://${SERVER_IP}"
echo "  注意: 浏览器需要接受自签名证书警告"
echo "=========================================="
```

使用方式：
1. 将脚本保存为 `deploy.sh`
2. 修改顶部的 `<YOUR_SERVER_IP>` 和密码
3. 上传项目代码到 `/opt/free4chat-elixir/`
4. `chmod +x deploy.sh && sudo ./deploy.sh`
